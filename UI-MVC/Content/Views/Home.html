Sustainable Solvents

Sussol is a research project by two departments of the Karel de Grote University College of Antwerp (Belgium), An easy to use solvent clusterer platform that is based on Machine Learning algorithms of the WeKa engine, We look for alternative sustainable solvents to replace solvents that are not biodegradable or carcinogenic, Using unsupervised learning our system models existing solvents into meaningful clusters, A new candidate solvent can then be offered to the models and will be classified into the most appropriate cluster, You can create your own organisation and share your results with your colleagues, The project is a collaboration between the Applied Informatics department and the Chemical department of Karel de Grote University College

Why Sussol?

  • The big problem

    Solvents are used in many industrial environments like cleaning products, paint and chemical solutions. Traditional Solvents can be toxical, flammable and not renewable. The searching process for alternatives to these traditional solvents requires a lot of time and people. Moreover, knowledge is needed about the chemical and physical properties of these solvents, but this knowledge isn't always available to the businesses.

  • Our Goal

    In this project we are working on a solvent substitution tool by applying Machine Learning algorithms to a two-dimensional matrix of solvents. Machine learning algorithms are able to cluster big amounts of multidimensional data in groups based on matching characteristics. With this tool companys can evaluate their new candidate solvents in an easy, less time-consuming way.

Explore the magic algorithms

K Means

The K Means algorithm is the mother of all clustering algorithms. It starts off with a pre-specified number of clustercenters and iteratively adapts the clusters based on the training data.

Canopy

The Canopy algorithm is intended to speed up clustering operations on large data sets. It works in 2 phases. In the first run it generates a rough clustering of the data. In the second run, the clusters are finetuned, mostly using the K Means algorithm.

X Means

The X Means algorithm is a variant on K Means, without a pre-specified number of clusters. It uses the Bayes Information Criterion to guide the choice of the best clustering model during training.

Expectation Maximization

The Expectancy Maximization algorithm is based on the statistics of the dataset. It is a good choice for datasets that contain many missing data. It clusters the dataset in random clusters and iteratively changes the clusters, trying to improve the log-likelihood of the resulting clusterset.

Self Organising Map

The Self Organising Map is an artificial neural network that clusters datasets by adapting the weights of the clustercenters. The algorithm is based on how real human neurons process visual information. It performs a topological mapping.

WeKa Engine

In this project we are working with the open source WeKa Machine Learning Engine. Weka is a collection of machine learning algorithms for data mining tasks and analyses. Weka contains tools for data pre-processing, classification, regression, clustering, association rules, and visualization.

Pre-processing

Data pre-processing is a very important step in the data mining process. It includes tasks like cleaning, normalization, selection, ... We use a data matrix containing 6070 solvents with each 48 physical and/or chemical, 25 (eco) toxologic, 19 CLP regulation symbols and 204 H/P/EUH features. This data set is cleaned and transformed into the correct WeKa format.

Data processing

Learning process

The pre-processed data is the training that is offered to 5 unsupervised learning algorithms : K Means, Canopy, X Means, the Expectancy Maximization algorithm and the Self Organising Map neural network.

Learning process

Learning Model

The result of the learning process is a ‘model’. A model is a knowledge representation that has been learned by the algorithm. It contains cluster and solvent information. The model can then be visualised showing the classification of the solvents of the training set.

Read more Model

Classify my own solvent

The model can also be used to offer a new solvent for classification. The solvent will be classified, using a specific algorithm, into the most applicable cluster. Additional information on the sustainability and other parameters of the new solvent can then be rendered, based on the cluster it is allocated to.
